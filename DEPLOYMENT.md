# Deployment Guide: Minds Gateway

This guide covers the deployment of the **Multi-AI Agents Communication Layer** (Minds Gateway) to **GCP Cloud Run**.

## 📋 Prerequisites

1.  **Google Cloud Project**: An active GCP project with billing enabled.
2.  **GCP Service Account**: A service account with the following roles:
    - `Cloud Datastore User` (for Firestore access)
    - `Cloud Run Invoker`
3.  **Firestore**: Ensure Firestore is initialized in your GCP project (Native Mode).
4.  **Telegram Bot**:
    - A Telegram Bot Token from [@BotFather](https://t.me/botfather).
    - The `Chat ID` of the Telegram group where notifications will be sent.
5.  **GitHub Token**: `ghp_...` (for pushing/pulling code).

## 🛠️ Step 1: Environment Configuration

Create a `.env` file (locally for testing) or set these in the **GCP Cloud Run Environment Variables**:

```env
TELEGRAM_BOT_TOKEN=your_bot_token_here
TELEGRAM_GROUP_CHAT_ID=your_group_chat_id_here
X_API_KEY=v-minds-secure-key-2026
GOOGLE_CLOUD_PROJECT=your-project-id
```

## 🐳 Step 2: Dockerize the Gateway

Create a `Dockerfile` in the root directory:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# Run uvicorn on port 8080 (Cloud Run default)
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8080"]
```

## 🚀 Step 3: Deploy to Cloud Run

Run the following commands from your local machine (ensure `gcloud` is authenticated):

1.  **Build the Image**:
    ```bash
    gcloud builds submit --tag gcr.io/YOUR_PROJECT_ID/minds-gateway
    ```

2.  **Deploy to Cloud Run**:
    ```bash
    gcloud run deploy minds-gateway \
      --image gcr.io/YOUR_PROJECT_ID/minds-gateway \
      --platform managed \
      --region asia-east1 \
      --allow-unauthenticated \
      --set-env-vars TELEGRAM_BOT_TOKEN=... \
      --set-env-vars TELEGRAM_GROUP_CHAT_ID=... \
      --set-env-vars X_API_KEY=...
    ```

## 🔗 Step 4: Connecting the Minds

To allow an external Mind bot (like @Radar_Scout) to communicate back:
1.  Provide the **Service URL** generated by Cloud Run (e.g., `https://minds-gateway-xyz.a.run.app`).
2.  The Mind bot must hit the following endpoint to report results:
    - **Endpoint**: `POST /missions/callback`
    - **Header**: `x-api-key: your_secure_key`
    - **Payload**:
      ```json
      {
        "mission_id": "uuid-string",
        "responder": "@Mind_Bot_Name",
        "result": "Structured task results here..."
      }
      ```

## 🧪 Step 5: Testing the Gateway

You can test the dispatching logic using `curl`:

```bash
curl -X POST "https://YOUR_URL/missions/dispatch" \
     -H "x-api-key: v-minds-secure-key-2026" \
     -H "Content-Type: application/json" \
     -d '{
       "sender": "Victor",
       "target": "@Radar_Scout",
       "task": "Test mission for NFT trends"
     }'
```
